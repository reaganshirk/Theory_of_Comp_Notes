Formal Definition for NFAs

  • An NFA (nondeterministic finite automaton/accepter) is defined by a 5-tuple (\(Q\), \(\Sigma\), \(\delta\), \(q_o\), \(F\)) where \(Q\) is a finite alphabet, \(q_o\) is a staring state, \(F\) is a set of accepting states, and \(\delta\) is a transition state equal to \((Q \times \Sigma) \cup \left\{\lambda\right\} \to P(Q)\)
  • \(Q, \Sigma, q_o, F\) are all the same as DFAs
    • \(\delta\): \(A \times \Sigma \to Q\) is the case for DFAs

NFA Example, \(\Sigma_{\lambda} = \left\{0, 1, \lambda\right\}\)

Proof

  • Definition: Two machine are equivalent iff they recognize the same language. In other words: \(L(M_1) = L(M_2)\)
  • Theorem: Every NFA has an equivalent DFA
  • Proof idea: Create DFA \(M\) that simulates the behavior of NFA \(N\). At any point in time we need to keep track on \(M\) of all of the states where \(N\) is currently at (\(N\) has a copy of itself in one of these states). All we need to do is define the 5-tuple
  • Actual proof:
    • Assume for know as lambda transitions
      • \(Q_m = P(Q_n)\)
        • We need one state in \(M\) for every one of the \(2^Q\) subsets in \(N\).
      • \(\Sigma_m = \Sigma_n\)
      • \(q_{om} = q_{on}\)
      • \(F_m = \left\{R \in P(Q) | \exists~r \in R~and~r \in F_n\right\}\)

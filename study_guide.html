Regular Languages

  • A language is called a regular language if some finite automaton recognizes it

Regular Operators

  • Like a tool for regular languages, regular operators are operations specifically designed for manipulating regular languages
  • Regular languages are defined as follows:
    • Let \(A\) and \(B\) be languages. We define regular operations union, concatenation, and star as:
      • Union: \(A \cup B = \left\{x | x \in A or x \in B\right\}\)
      • Concatenation: \(A\circ B = \left\{xy | x \in A ~\text{and}~ y \in B\right\}\)
      • Star: \(A^* = \left\{x_1x_2...x_k | k \geq 0 ~\text{and each}~ x_i \in A\right\}\)
  • Union takes all of the strings in both \(A\) and \(B\) and lumps them together into one language
  • Concatenation attaches a string from \(A\) onto the front of a string from \(B\) in all possible ways to get the strings in the new language
  • Star is a unary operation that attaches any number of strings in \(A\) together to get a string in a new language. It includes \(0\) as a possibility and the empty string \(\lambda\) is always a member of the language

Closure Under Regular Operators

  • Union: The class of regular languages is closed under the union operation. In other words, if \(A_1\) and \(A_2\) are regular languages, then so is \(A_1 \cup A_2\)
    • Proof: Let \(M_1\) recognize \(A_1\) where \(M_1 = \left(Q_1, \Sigma, \delta_1, q_1, F_1\right)\) and \(M_2\) recognize \(A_2\) where \(M_2 = \left(Q_2, \Sigma, \delta_2, q_2, F_2\right)\)
      • Construct \(M\) to recognize \(A_1 \cup A_2\) where \(M = \left(Q, \Sigma, \delta, q_o, F\right)\)
        • \(Q = \left\{(r_1, r_2) | r_1 \in Q_1 ~\text{and}~ r_2 \in Q_2\right\}\)
          This is the cartesian product of sets \(Q_1\) and \(Q_2\), written as \(Q_1 \times Q_2\). It is the set of all pairs of states, the first from \(Q_1\) and the second from \(Q_2\).
        • \(\Sigma\), the alphabet, is the same as in \(M_1\) and \(M_2\). In this theorem (and in all subsequent similar theorems) we assume for simplicity that both \(M_1\) and \(M_2\) have the same input alphabet \(\Sigma\). The theorem remains true if they have different alphabets, you would just have to modify the proff to let \(\Sigma = \Sigma_1 \cup \Sigma_2\)
        • \(\delta\), the transition function, is defined as follows:
          For each \((r_1, r_2) \in Q\) and each \(a \in \Sigma\), let
          \(\delta((r_1, r_2), a) = \left(\delta_1(r_1, a), \delta_2(r_2, a)\right)\)
          Hence \(\delta\) gets a state of \(M\) (which is actually a pair of states from \(M_1\) and \(M_2\)), together with an input symbol, and returns \(M's\) next state.
        • \(q_o\) is the pair \((q_1, q_2)\)
        • \(F\) is the set of paris in which either member is an accept state of \(M_1\) or \(M_2\). We can write it as
          \(F = \left\{(r_1, r_2) | r_1 \in F ~\text{or}~ r_2 \in F_2\right\}\)
          This expression is the same as \(F = (F_1 \times Q_2) \cup (Q_1 \times F_2)\)
    • This is the entire construction for the machine \(M\) that recognizes \(A_1 \cup A_2\)
  • Concatenation: The class of regular languages is closed under the concatenation operation. In other words, if \(A_1\) and \(A_2\) are regular languages then so is \(A_1 \circ A_2\).
    • This is similar to the proof above. However, instead of constructing a machine that will accept if either \(M_1\) or \(M_2\) accept, it must accept if the input can be broken into two pieces, one of which will be accepted by \(M_1\) and the other which will be accepted by \(M_2\). The issue arises with not knowing where to break the input; we need nondeterminism to solve this (jump to NFAs).

Designing a Finite Automaton

  • The goal is to design a machine that will recognize the language \(L = \left\{w | w~contains~an~odd~number~of~ones\right\}\) in the alphabet \(\Sigma = \left\{0, 1\right\}\)
  • All you need to remember is whether the number of ones seen sof ar is even or odd
    • If you read a \(1\), flip states. If you read a \(0\), stay in the same state
  • Two states needed:
  • Assign transitions by seeing how to go from one possibility to another based on reading a symbol
    • i.e. if you’re in \(q_{even}\) and you read a \(1\), go to \(q_{odd}\) and vice versa
    • if you’re in either state and you read a \(0\), stay there
  • Determine which will be your starting and accepting states.
    • You want to have an odd number of ones so the accept state should be \(q_{odd}\)
    • The starting state should be \(q_{even}\) because having no ones would be an even number of them.
